39  	 * Keep connection alive.
40  	 * 
41  	 * <p>
42  	 * The value of this socket option is a {@code Boolean} that represents
43  	 * whether the option is enabled or disabled. When the {@code SO_KEEPALIVE}
44  	 * option is enabled the operating system may use a <em>keep-alive</em>
45  	 * mechanism to periodically probe the other end of a connection when the
46  	 * connection is otherwise idle. The exact semantics of the keep alive
47  	 * mechanism is system dependent and therefore unspecified.
48  	 * 
49  	 * <p>
50  	 * The initial value of this socket option is {@code FALSE}. The socket
51  	 * option may be enabled or disabled at any time.
52  	 * 
53  	 * @see <a href="http://www.ietf.org/rfc/rfc1122.txt">RFC&nbsp;1122 *
54  	 *      Requirements for Internet Hosts -- Communication Layers< /a>
55  	 * @see Socket#setKeepAlive
56  	 */

59  	/**
60  	 * The size of the socket send buffer.
61  	 * 
62  	 * <p>
63  	 * The value of this socket option is an {@code Integer} that is the size of
64  	 * the socket send buffer in bytes. The socket send buffer is an output
65  	 * buffer used by the networking implementation. It may need to be increased
66  	 * for high-volume connections. The value of the socket option is a
67  	 * <em>hint</em> to the implementation to size the buffer and the actual
68  	 * size may differ. The socket option can be queried to retrieve the actual
69  	 * size.
70  	 * 
71  	 * <p>
72  	 * For datagram-oriented sockets, the size of the send buffer may limit the
73  	 * size of the datagrams that may be sent by the socket. Whether datagrams
74  	 * larger than the buffer size are sent or discarded is system dependent.
75  	 * 
76  	 * <p>
77  	 * The initial/default size of the socket send buffer and the range of
78  	 * allowable values is system dependent although a negative size is not
79  	 * allowed. An attempt to set the socket send buffer to larger than its
80  	 * maximum size causes it to be set to its maximum size.
81  	 * 
82  	 * <p>
83  	 * An implementation allows this socket option to be set before the socket
84  	 * is bound or connected. Whether an implementation allows the socket send
85  	 * buffer to be changed after the socket is bound is system dependent.
86  	 * 
87  	 * @see Socket#setSendBufferSize
88  	 */

91  	/**
92  	 * The size of the socket receive buffer.
93  	 * 
94  	 * <p>
95  	 * The value of this socket option is an {@code Integer} that is the size of
96  	 * the socket receive buffer in bytes. The socket receive buffer is an input
97  	 * buffer used by the networking implementation. It may need to be increased
98  	 * for high-volume connections or decreased to limit the possible backlog of
99  	 * incoming data. The value of the socket option is a <em>hint</em> to the
100 	 * implementation to size the buffer and the actual size may differ.
101 	 * 
102 	 * <p>
103 	 * For datagram-oriented sockets, the size of the receive buffer may limit
104 	 * the size of the datagrams that can be received. Whether datagrams larger
105 	 * than the buffer size can be received is system dependent. Increasing the
106 	 * socket receive buffer may be important for cases where datagrams arrive
107 	 * in bursts faster than they can be processed.
108 	 * 
109 	 * <p>
110 	 * In the case of stream-oriented sockets and the TCP/IP protocol, the size
111 	 * of the socket receive buffer may be used when advertising the size of the
112 	 * TCP receive window to the remote peer.
113 	 * 
114 	 * <p>
115 	 * The initial/default size of the socket receive buffer and the range of
116 	 * allowable values is system dependent although a negative size is not
117 	 * allowed. An attempt to set the socket receive buffer to larger than its
118 	 * maximum size causes it to be set to its maximum size.
119 	 * 
120 	 * <p>
121 	 * An implementation allows this socket option to be set before the socket
122 	 * is bound or connected. Whether an implementation allows the socket
123 	 * receive buffer to be changed after the socket is bound is system
124 	 * dependent.
125 	 * 
126 	 * @see <a href="http://www.ietf.org/rfc/rfc1323.txt">RFC&nbsp;1323: TCP *
127 	 *      Extensions for High Performance< /a>
128 	 * @see Socket#setReceiveBufferSize
129 	 * @see ServerSocket#setReceiveBufferSize
130 	 */

133 	/**
134 	 * Re-use address.
135 	 * 
136 	 * <p>
137 	 * The value of this socket option is a {@code Boolean} that represents
138 	 * whether the option is enabled or disabled. The exact semantics of this
139 	 * socket option are socket type and system dependent.
140 	 * 
141 	 * <p>
142 	 * In the case of stream-oriented sockets, this socket option will usually
143 	 * determine whether the socket can be bound to a socket address when a
144 	 * previous connection involving that socket address is in the
145 	 * <em>TIME_WAIT</em> state. On implementations where the semantics differ,
146 	 * and the socket option is not required to be enabled in order to bind the
147 	 * socket when a previous connection is in this state, then the
148 	 * implementation may choose to ignore this option.
149 	 * 
150 	 * <p>
151 	 * For datagram-oriented sockets the socket option is used to allow multiple
152 	 * programs bind to the same address. This option should be enabled when the
153 	 * socket is to be used for Internet Protocol (IP) multicasting.
154 	 * 
155 	 * <p>
156 	 * An implementation allows this socket option to be set before the socket
157 	 * is bound or connected. Changing the value of this socket option after the
158 	 * socket is bound has no effect. The default value of this socket option is
159 	 * system dependent.
160 	 * 
161 	 * @see <a href="http://www.ietf.org/rfc/rfc793.txt">RFC&nbsp;793: *
162 	 *      Transmission Control Protocol< /a>
163 	 * @see ServerSocket#setReuseAddress
164 	 */

166 			"SO_REUSEADDR", Boolean.class);
167 	/**
168 	 * Linger on close if data is present.
169 	 * 
170 	 * <p>
171 	 * The value of this socket option is an {@code Integer} that controls the
172 	 * action taken when unsent data is queued on the socket and a method to
173 	 * close the socket is invoked. If the value of the socket option is zero or
174 	 * greater, then it represents a timeout value, in seconds, known as the
175 	 * <em>linger interval</em>. The linger interval is the timeout for the
176 	 * {@code close} method to block while the operating system attempts to
177 	 * transmit the unsent data or it decides that it is unable to transmit the
178 	 * data. If the value of the socket option is less than zero then the option
179 	 * is disabled. In that case the {@code close} method does not wait until
180 	 * unsent data is transmitted; if possible the operating system will
181 	 * transmit any unsent data before the connection is closed.
182 	 * 
183 	 * <p>
184 	 * This socket option is intended for use with sockets that are configured
185 	 * in {@link java.nio.channels.SelectableChannel#isBlocking() blocking} mode
186 	 * only. The behavior of the {@code close} method when this option is
187 	 * enabled on a non-blocking socket is not defined.
188 	 * 
189 	 * <p>
190 	 * The initial value of this socket option is a negative value, meaning that
191 	 * the option is disabled. The option may be enabled, or the linger interval
192 	 * changed, at any time. The maximum value of the linger interval is system
193 	 * dependent. Setting the linger interval to a value that is greater than
194 	 * its maximum value causes the linger interval to be set to its maximum
195 	 * value.
196 	 * 
197 	 * @see Socket#setSoLinger
198 	 */
199 	public static final SocketOption<Integer> SO_LINGER = new SocketOption<Integer>(
200 			"SO_LINGER", Integer.class);
201 	/**
202 	 * Disable the Nagle algorithm.
203 	 * 
204 	 * <p>
205 	 * The value of this socket option is a {@code Boolean} that represents
206 	 * whether the option is enabled or disabled. The socket option is specific
207 	 * to stream-oriented sockets using the TCP/IP protocol. TCP/IP uses an
208 	 * algorithm known as <em>The Nagle Algorithm</em> to coalesce short
209 	 * segments and improve network efficiency.
210 	 * 
211 	 * <p>
212 	 * The default value of this socket option is {@code FALSE}. The socket
213 	 * option should only be enabled in cases where it is known that the
214 	 * coalescing impacts performance. The socket option may be enabled at any
215 	 * time. In other words, the Nagle Algorithm can be disabled. Once the
216 	 * option is enabled, it is system dependent whether it can be subsequently
217 	 * disabled. If it cannot, then invoking the {@code setOption} method to
218 	 * disable the option has no effect.
219 	 * 
220 	 * @see <a href="http://www.ietf.org/rfc/rfc1122.txt">RFC&nbsp;1122: *
221 	 *      Requirements for Internet Hosts -- Communication Layers< /a>
222 	 * @see Socket#setTcpNoDelay
223 	 */
224 	public static final SocketOption<Boolean> TCP_NODELAY = new SocketOption<Boolean>(
225 			"TCP_NODELAY", Boolean.class);
